# Anorthoscope Simulator — Web (TypeScript + Canvas)

p5.js / Processing 版の作り直し。素の Canvas 2D + TypeScript で、依存ライブラリなし。

## 開発

```bash
cd web
npm install
npm run dev      # http://localhost:5199
```

## ビルド（GitHub Pages 用）

```bash
npm run build    # リポジトリ直下 docs/ に出力
```

## 画像の追加

`web/public/data/` に png/jpg/gif を置き、`manifest.json` を更新する:

```bash
cd web/public/data
ls | grep -iE '\.(png|jpe?g|gif)$' | sort \
  | python3 -c "import sys,json;print(json.dumps({'images':[l.strip() for l in sys.stdin]},indent=2))" \
  > manifest.json
```

実行中に UI の「+」ボタンからローカル画像を一時追加することも可能。

## 設計メモ

- **内部解像度の頭打ち** (`config.ts` の `STAGE_MAX_HEIGHT`): 表示は CSS で拡大し、
  内部レンダリングは常にこの高さ以下。プロジェクタの高解像度でもメモリ・GPU 負荷が
  解像度² で膨張しない（旧 Processing 版が Intel Mac + プロジェクタで落ちた主因への対策）。
- **オフスクリーン Canvas の使い回し**: 毎フレームの `getImageData` / 新規確保をやめ、
  ストリップ抽出は `drawImage` のソース矩形クロップで行う。
- **delta-time**: `requestAnimationFrame` の経過時間で角度を進め、フレームレート非依存。

## 構成

```
src/
  main.ts              起動・状態・メインループ・各種の配線
  config.ts            定数とパラメータ型
  images.ts            画像読み込み（ImageBitmap）
  gallery.ts           ブラウザ内ギャラリー（localStorage）
  engine/Simulation.ts 描画エンジン（2パネル + スリット抽出）
  ui/Controls.ts       下部コントロールバー（DOM・ボタン + 数値入力）
  ui/Guide.ts          操作ガイドのオーバーレイ
  ui/PaintEditor.ts    ペイントモード（扇形描画 → 360°展開）
  ui/CompressEditor.ts 圧縮モード（360°画像 → 1/K 扇形、ペイントの逆）
  ui/Gallery.ts        保存作品の一覧（使う/編集/削除）
  engine/wedge.ts       扇形⇔360°の極座標変換（両エディタで共有）
```

## 操作

キーボード操作は廃止し、すべて画面下部のボタンと数値入力で行う（ESC は開いている
オーバーレイを閉じるためだけに使用）。

- **◀ ▶ / サムネイル**: 画像切替（前・現在・次の3枚のみ表示）
- **⏸ / ▶**: 再生 / 停止
- **数値入力**: スリット数・速度・画像回転係数・スリット回転係数
- **Line**: 右パネルの赤ガイドライン切替
- **🖌 ペイント**: お絵かきモードを開く
- **🖼 ギャラリー**: 保存した絵を呼び出す / 編集 / 削除
- **＋画像**: 画像ファイルを追加
- **⛶**: フルスクリーン

## ペイントモード

- 円を **2〜8 分割**したうちの **1 扇形（黄色枠）内だけ**に描ける。
- 描いた扇形は **角度方向へ引き伸ばして 360° を埋める**（コピー配置ではなく、極座標の
  角度ストレッチ。歪んだ 1 枚のアナモルフォーズ画像になる）。エディタ右側に展開プレビュー。
- ツール: ブラシ / 消しゴム / 直線 / 円 / 塗りつぶし / 元に戻す / 全消し（線色・太さ可変）。
- **背景色**を1色から選べる（既定は黒）。描画レイヤとは別に保持され、再編集でも変更可。
- **＋ファイルから写真を読み込める。写真は本体コンテンツ**（トレース台紙ではない）。
  **✋ツールでドラッグして移動、－／＋で拡大縮小**できる（扇形にクリップされる）。
  写真は手描きレイヤーとは独立しており、元に戻す/全消しは手描き部分だけに効く
  （「写真を消す」ボタンで写真だけ削除も可能）。
  **保存すると写真の実体データ・位置・拡大率もそのまま保存され、ギャラリーから
  再編集すると写真も含めて完全に復元される。**
- **保存**すると展開画像が PNG 化されブラウザ内ギャラリーに入り、そのままシミュレータの
  画像として再生できる。ギャラリーから再編集も可能。

## 圧縮モード（ペイントモードの逆）

- 任意の **360° 画像**（画像一覧から選ぶ、またはファイルから）を読み込み、**1/2〜1/16**
  の任意の扇形へ圧縮する。アノルソスコープで見える像と同じ幾何（`engine/wedge.ts` の
  `fullToWedge`）だが、ペイントモードと違って**隙間を埋める必要はない**（全周の内容が
  そのまま 1/K の扇形へ畳み込まれるだけ）。
- 左に元の360°画像、右に黄色枠の圧縮結果（1/K 扇形）を表示。
- **左の元画像に、ペイントモードと同じツール**（ブラシ / 消しゴム / 直線 / 円 /
  塗りつぶし / 元に戻す / 消去、色・太さ可変）で直接描き足せる。**左に描くと右の
  圧縮結果へリアルタイムに反映**される（分割数を変えた時も同様に即再計算）。
  手描き加筆は読み込んだ写真とは別レイヤーなので、元に戻す/消去は加筆分だけに効く。
- ライブプレビューは低解像度（300px）でリアルタイム再計算し、保存時のみフル解像度
  （800px）で再圧縮する（`schedulePreview()` の rAF デバウンス、ペイントモードと同じ手法）。
- 保存すると（元画像＋手描き加筆を合成してフル解像度で圧縮した結果が）ギャラリーに入り、
  シミュレータの画像として使える。
